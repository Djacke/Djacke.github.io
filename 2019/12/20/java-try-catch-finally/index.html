try block
try block 包含正常情况下运行的应用程序代码，比如，读取文件，读写数据库或者复杂的业务代码，由一对大括号包含应用代码。

catch block
catch block 位于try block后，必须处理 try block 抛出的受检查异常或者未受检查异常。

finally block
finally block 给我们机会在每次执行完try-catch块后运行我们要执行的代码，无论有没有异常发生

对于可关闭资源，比如IO流，可用try-with-resources语句处理异常，在这种方式下，我们不需要手动关闭流，JVM会替我们完成关闭流的工作，从而省去了finally block的编写。

要分析清楚如上方法的返回值，需要想搞清楚几个概念，Java方法实在栈帧中执行，栈帧是线程私有的栈的单位，执行方法的线程会为每一个方法分配一小块空间来作为执行是的内存空间，栈帧分为四个区域：

  • 操作数栈 用来保存正在执行的表达式中的操作数
  • 局部变量区 用来保存方法中使用的变量，包括方法的参数，方法内部生命的变量，以及方法中使用到的对象的成员变量或者类的变量（静态变量），最后两种变量恢复制到局部变量区，即多线程下变量不共享。
  • 动态链接 栈帧持有一个指向方法区常量池中所属方法的引用。这个引用是为了支持动态连接。动态连接是指符号引用在运行时转化为直接引用。
  • 方法返回地址 方法的退出有两种方式，正常执行结束退出和异常退出。异常退出一般根据异常处理表来确定返回地址，栈帧中不会存储这一部分信息。方法的退出实际上就是将栈帧从栈里弹出，恢复上一层栈帧的本地变量表和操作数栈，并且将返回值压入上一栈帧的操作数栈。

了解了栈帧的相关概念，我们详细分析下方法的返回过程，首先，try块中有return出现时，执行完try i的值就已经确定了，catch 块中的return会覆盖try块中的return，同样finally中的return也会覆盖以上两个块中的返回值

我们以Demo_1为例分析如下：

  • return时i的值为1，将1 保存在操作数栈顶
  • 再将操作数栈顶的值复制到局部变量区作为返回值
  • 再执行finally中的i++
  • 再将局部变量区得知复制到操作数栈顶
  • return 返回操作数栈顶的值
